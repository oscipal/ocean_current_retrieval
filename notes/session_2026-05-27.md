# Session notes — 2026-05-27

End-to-end push to characterise the GAMMA pipeline against drifters and the
Sentinel-1 OCN L2 product, plus several new estimator / wave-bias options.

## Overview

Goals tackled this session:

1. Build a **method-sweep validation** so we can see the contribution of
   every processing step (Doppler estimator, wave bias, descallop,
   mispointing) on the 68-fix drifter set.
2. Understand why the original headline scatter showed **r ≈ 0.06** for the
   custom pipeline.
3. Add a **CDOP-2** wave-Doppler model and test whether it moves the needle.
4. Investigate deramping in detail (pixel values, Doppler spectra, custom vs
   GAMMA deramp, OCN's `rvlDcObs`).
5. Add an **FFT spectral-centroid** Doppler estimator that matches OCN's
   per-cell variability better than GAMMA's lag-1 averaging.
6. Switch the drifter validation to use **pure OCN `rvlRadVel`** as the
   comparison baseline (not the hybrid that re-derived currents from OCN's
   `f_dc` through our pipeline).

## Key findings

### Why r was 0.06

- The old `v_los_s1` column = method `04_stokes_ocn_desc_last` in the sweep:
  bias = −0.025 (essentially zero) but **r = +0.04**.
- `wave_source='ocn'` (= OCN `owiRadVel`) is empirically calibrated against
  drifters/reanalysis, so it absorbs the wind-driven surface current we are
  trying to retrieve, **not just the wave Doppler**. Subtracting it kills
  the variance that correlates with the drifter.
- Across 60+ scenes × 1 fix each, the only "signal" r can measure is
  scene-to-scene current variation (std ≈ 0.33 m/s). Per-scene retrieval
  noise from the bias-killing config is ≈ 0.4 m/s, which swamps the signal.
- Switching back to `wave_source='mouche'` recovers **r ≈ 0.57**, matching
  the operational OCN product at the cost of a small constant offset.

### Method-sweep ranking (68 fixes, 18 methods)

| | bias | RMSE | r |
|---|---|---|---|
| `05_full_mouche_last` (Mouche + descallop + mispointing) | −0.35 | 0.55 | **+0.57** |
| `ocn_product` (hybrid; rvlDcObs + our corrections + mispointing) | −0.31 | 0.48 | **+0.58** |
| `glo12` (model ceiling) | 0.00 | 0.17 | **+0.85** |
| `05_full_ocn_last` (= old `v_los_s1_ocn`) | +0.19 | 0.46 | +0.08 |
| `04_stokes_ocn_desc_last` (= old `v_los_s1`) | −0.02 | 0.41 | +0.04 |
| any `_first` (mosaic-first) variant | mixed | wide | **negative** — branch is broken |

### Tighter drifter matching

Slicing the validation CSV by `dt_min`:

| dt threshold | CDOP `v_los_s1_ocn` r | OCN-product r |
|---|---|---|
| ≤ 30 min | +0.56 | +0.58 |
| ≤ 20 min | +0.59 | +0.64 |
| ≤ 15 min | **+0.64** | +0.56 |
| ≤ 10 min | **+0.69** | +0.66 |

**At Δt ≤ 15 min the CDOP-driven custom pipeline overtakes OCN on r.**
Real co-location matters: at ±30 min the drifter has drifted ~600 m and
inertial/wind transients add noise to the comparison.

### Deramping — what happens, what doesn't

Pixel values (full IW1 image, 320 M pixels):

| | dtype | amp min | amp max | amp mean | amp std |
|---|---|---|---|---|---|
| Original TOPS TIFF | `complex_int16` | 0 | 7 360 | 52.7 | 33.5 |
| GAMMA deramped SLC | `complex_float32` | 0 | 23.6 | 0.15 | 0.11 |

Ratio ≈ **1/300** — this is `par_S1_SLC`'s β₀ radiometric calibration,
**not** the deramp. `SLC_deramp_ScanSAR` is phase-only.

Block-wise f_dc (256×256, full IW1, lag-1 autocorrelation):

| | min | max | mean | std |
|---|---|---|---|---|
| pre-deramp | −243 | +243 | −1.9 | **130 Hz** |
| post-deramp | −23 | +33 | +4.3 | **3.5 Hz** |

Bandwidth collapses by ~37× — that is the deramp doing its job. Without it
the signal is aliased white noise across the full PRF.

### Custom deramp vs GAMMA deramp (burst 5, IW1)

| method | min | max | mean | std |
|---|---|---|---|---|
| custom `current` | +18.7 | +28.4 | +22.6 | 1.7 Hz |
| custom `esa_eq1` | +16.6 | +30.0 | +22.4 | 2.6 Hz |
| GAMMA `SLC_deramp_ScanSAR` | +0.6 | +7.6 | +3.7 | **1.4 Hz** |

Both kill the chirp; GAMMA's also centres the signal closer to zero. The
~+19 Hz residual in the custom path is later cleaned up by descalloping —
the per-burst `engine='ours'` pipeline doesn't notice because that constant
gets removed downstream.

### Doppler estimator alternatives (OCN grid 233×128)

| | std (Hz) | range (Hz) |
|---|---|---|
| lag-1 (current, GAMMA-style) | 3.7 | −32 → +42 |
| lag-1 on 64×16 sub-window | 5.1 | −34 → +30 |
| FFT peak per cell | **42** | −160 → +167 (too noisy) |
| **FFT spectral centroid** | **10.3** | −47 → +52 |
| **OCN `rvlDcObs`** (reference) | **12.7** | −94 → +181 |

FFT spectral centroid almost matches OCN's per-cell variability. Lag-1
averages too much (10 k complex pairs per cell) → smooth field; FFT peak
is too coarse (8 Hz bins at 64-point FFT) → spurious extremes.

### Three-panel `v_current` comparison (visualisation config)

`lag1_gamma`, `fft_centroid`, OCN-rvlDcObs-through-our-pipeline all run
through the **same downstream corrections** (Mouche + no descallop + blend
+ 3×1 smoothing + with mispointing):

| panel | std (m/s) | range (m/s) |
|---|---|---|
| `lag1_gamma · viz config` (mispt) | 0.243 | −0.66 → +0.67 |
| `fft_centroid · viz config` (mispt) | 0.169 | −0.39 → +1.17 |
| Sentinel-1 OCN (mispt) | 0.332 | −3.70 → +2.66 |

Mispointing centres lag1_gamma exactly (its f_dc had a +0.30 m/s bias that
mispointing cancels) and **over-corrects** fft_centroid (its f_dc was
already centred, so mispointing now pushes it positive). OCN's wider tails
reflect its finer effective grid.

## Code changes

### New files

- `scripts/sentinel_1/cdop.py` — verbatim CDOP NN from
  `nansencenter/openwind` (GPL-3, Mouche 2012). VV + HH coefficients.
- `scripts/validation/run_method_sweep.py` — long-format CSV sweep over 18
  methods (4 GAMMA runs + 1 OCN-DC run per scene-subswath), pickle-cached.
- `scripts/validation/derive_ocn_dc_chain.py` — derive 4 incremental
  methods on the OCN-DC side from the existing cached pickles, no recompute.
- `scripts/poster/make_method_scatter.py` — small-multiples scatter, one
  panel per method, with `--only-mosaic` / `--only-stage` filters.
- `scripts/poster/make_dop_comparison.py` — three-panel SVG comparing
  `lag1_gamma` / `fft_centroid` / OCN with `--field {v_current, f_dc}`.

### Modified files

- `scripts/sentinel_1/pipeline.py`
  - `run_gamma_pipeline_from_safe`: new params `mosaic_mode`, `f_dc_method`
    (`'lag1_gamma' | 'fft_centroid'`), `fft_block_az`, `fft_block_rg`.
  - `geom_source` defaults per mode (gamma for last+lag1, poeorb for first
    or fft_centroid).
  - Wave-source branch in `run_gamma_dop2d_pipeline` now accepts `'cdop'`.
- `scripts/sentinel_1/gamma_variants.py`
  - New `fft_centroid_doppler(...)` — drop-in alternative to
    `doppler_2d_SLC` at any block size (default 64×168 = OCN grid).
- `scripts/sentinel_1/metocean.py`
  - New `compute_wave_doppler_bias_cdop(...)` calling the CDOP NN.
- `scripts/validation/run_drifter_validation.py` and `_all.py`
  - Defaults flipped to the new standard config: `engine='gamma'`,
    `--gamma-descallop` ON by default (use `--no-gamma-descallop` to
    disable).
  - `print_summary` now includes `r`.
  - **Headline column is `v_los_s1_ocn`** (custom pipeline + OCN
    mispointing).
  - New `v_los_ocn_native` column = ESA's `rvlRadVel` sampled directly.
    Dropped the old hybrid `v_los_ocn_product`. Removed the per-burst
    `run_all_bursts(use_ocn_dc=True)` call — big speedup.
  - `--gamma-wave-source` now accepts `cdop` alongside `mouche` and `ocn`.
- `scripts/poster/make_scatter_figure.py`
  - Three series: Custom pipeline (`v_los_s1_ocn`), Sentinel-1 OCN
    (`v_los_ocn_native`), GLO12 model (`v_los_glo12`).

### Memory notes updated
